# AZ-Bench analysis pipeline

End-to-end recipe for going from a raw tournament `results.csv`,
per-trial wall-clock times, and 48 transcript files to the eleven
publication-ready figures used in the paper.

By default the BT fit does **not** deduplicate replicate games — every
row in `results.csv` is treated as an independent observation.

## Quick start

```bash
pip install -r requirements.txt
./run_all.sh
```

`run_all.sh` runs every command below in sequence and produces the
eleven figures in `outputs/pdfs/`. The README and `run_all.sh` are
kept in sync — if you edit one, edit the other.

---

## Step 1 — Compute BT ratings

```bash
python3 scripts/bt.py \
    --results input/results.csv \
    --output  outputs/bt_ratings.json
```

Prints (a) full / non-probe / probe-only BT leaderboards, (b) a
W/L/D leaderboard, (c) the per-agent summary table, (d) the
record-vs-Pons table — and writes `bt_ratings.json` for the
figure scripts to consume.

Ratings are anchored on the Pascal Pons solver at 2000 (a 400-point
spread = 10× win-probability ratio), so a rating is directly
interpretable as distance from game-theoretic optimality.

---

## Step 2 — BT-derived figures

These four scripts read `outputs/bt_ratings.json`:

```bash
python3 scripts/best_bt_lollipop.py    --ratings outputs/bt_ratings.json --out outputs/pdfs/best_bt_lollipop.pdf
python3 scripts/probe_bt.py            --ratings outputs/bt_ratings.json --out outputs/pdfs/probe_bt.pdf
python3 scripts/main_bt.py             --ratings outputs/bt_ratings.json --out outputs/pdfs/main_bt.pdf
python3 scripts/mean_bt_vs_release.py  --ratings outputs/bt_ratings.json --out outputs/pdfs/mean_bt_vs_release.pdf
```

---

## Step 3 — Input-derived figures

`move_time_bt.py` reads `results.csv`; the first-mover and budget
scripts read `results.csv` or `player_times_seconds.txt` directly.

```bash
python3 scripts/move_time_bt.py        --results input/results.csv               --out outputs/pdfs/move_time_bt.pdf

python3 scripts/first_mover_optimal.py --results input/results.csv --plot bars    --success-type wins-only --out outputs/pdfs/first_mover_optimal_bars_wins_only.pdf
python3 scripts/first_mover_optimal.py --results input/results.csv --plot release --success-type wins-only --out outputs/pdfs/first_mover_optimal_release_wins_only.pdf

python3 scripts/budget_3group.py       --times input/player_times_seconds.txt    --out outputs/pdfs/budget_3group.pdf
python3 scripts/budget_4main.py        --times input/player_times_seconds.txt    --out outputs/pdfs/budget_4main.pdf
```

`budget_3group.py` shows GPT-5.4 main vs the eval/non-eval pooled
probes. `budget_4main.py` (new) is the same style applied to the four
main agent setups: Opus 4.7, Opus 4.6, Gemini 3.1 Pro, and GPT-5.4 main.

---

## Step 4 — Transcript-derived figures

First build the keyword-rate CSV that both transcript figures consume:

```bash
python3 scripts/transcript_analysis.py input/transcripts \
    -p output.txt -o outputs/transcript_analysis.csv
```

Then plot:

```bash
python3 scripts/plot_heatmap.py outputs/transcript_analysis.csv \
    -o outputs/pdfs/keyword_heatmap

python3 scripts/plot_top4.py outputs/transcript_analysis.csv \
    --out outputs/pdfs/eval_vs_noneval_top4.pdf
```
